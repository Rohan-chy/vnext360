# Contributing Guide

## Commit Message Convention

This project follows a Conventional Commits–style format:


### Examples
- feat(auth): add login API
- fix(ui): button alignment issue
- docs: update README
- deps: upgrade redux toolkit

---

## Allowed Commit Types

- feat:       New features
- fix:        Bug fixes
- setup:      Initial setup or major structure changes
- deps:       Dependency updates
- style:      Formatting only (no logic changes)
- refactor:   Code restructuring
- revert:     Revert previous commits
- config:     Environment or config changes
- perf:       Performance improvements
- docs:       Documentation updates
- test:       Test updates
- build:      Build system changes (Vite, Webpack, Docker)
- ci:         CI/CD changes

---

## Rules
- Use **present tense**
- Keep the description short and clear
- One logical change per commit
